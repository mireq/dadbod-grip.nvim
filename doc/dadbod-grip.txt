1. INTRODUCTION                                          *grip-introduction*

dadbod-grip.nvim turns Neovim into a DataGrip-style database client powered
by vim-dadbod. It renders query results as editable grids with inline cell
editing, visual change staging, sort/filter/pagination, foreign key
navigation, schema browsing, and a SQL query pad.

Works standalone (no vim-dadbod-ui needed) or alongside it.

Quick start: >
  :GripConnect                  pick or add a database connection
  :GripSchema                   browse tables with columns and types
  :Grip users                   open a table in an editable grid
  :GripQuery                    open a SQL scratch pad
<

==============================================================================
2. REQUIREMENTS                                          *grip-requirements*

- Neovim 0.10 or later
- vim-dadbod (https://github.com/tpope/vim-dadbod)
- One or more database CLI tools in PATH:

  PostgreSQL ......... psql
  SQLite ............. sqlite3
  MySQL/MariaDB ...... mysql (8.0.3+ for --csv, or MariaDB 10.5+)
  DuckDB ............. duckdb

3. INSTALLATION                                              *grip-install*

lazy.nvim (recommended): >lua
  {
    "joryeugene/dadbod-grip.nvim",
    dependencies = { "tpope/vim-dadbod" },
    cmd = { "Grip", "GripSchema", "GripTables", "GripQuery", "GripConnect" },
    keys = {
      { "<leader>lg", function() require("dadbod-grip").open_smart() end,
        desc = "Grip: Open grid" },
      { "<leader>gs", "<cmd>GripSchema<cr>", desc = "Grip: Schema browser" },
      { "<leader>gt", "<cmd>GripTables<cr>", desc = "Grip: Table picker" },
      { "<leader>gq", "<cmd>GripQuery<cr>", desc = "Grip: Query pad" },
      { "<leader>gc", "<cmd>GripConnect<cr>", desc = "Grip: Connect" },
    },
    opts = {},
  }
<
Recommended extras: >lua
  -- SQL completion in query pad (auto-completes table/column names)
  { "kristijanhusak/vim-dadbod-completion", ft = { "sql" } }

  -- Better picker UX (optional, grip falls back to vim.ui.select)
  { "nvim-telescope/telescope.nvim" }  -- or { "ibhagwan/fzf-lua" }
<
packer.nvim: >lua
  use {
    "joryeugene/dadbod-grip.nvim",
    requires = { "tpope/vim-dadbod" },
  }
<

4. CONFIGURATION                                              *grip-config*

setup() is called automatically by the plugin loader with sensible defaults.
Override if needed: >lua
  require("dadbod-grip").setup({
    limit         = 100,    -- default row limit for SELECT queries
    max_col_width = 40,     -- max display width per column
    timeout       = 10000,  -- query timeout in milliseconds
  })
<
No default keymaps are set outside the grip buffer.

5. COMMANDS                                                 *grip-commands*

All commands are global and available after the plugin loads.

                                                                     *:Grip*
:Grip [table|SQL|file|URL]
  Open a table in an editable grid, run a SQL query, open a local file,
  or query a remote file via DuckDB httpfs.

  Examples: >
    :Grip users
    :Grip SELECT * FROM orders LIMIT 50
    :Grip /path/to/data.parquet
    :Grip https://example.com/data.csv
<

                                                               *:GripSchema*
:GripSchema
  Toggle the schema browser sidebar. Shows tables and views with expandable
  columns, type abbreviations, and PK/FK markers.

                                                               *:GripTables*
:GripTables
  Open the table picker. Uses telescope.nvim if available, then fzf-lua,
  then falls back to vim.ui.select. Shows column preview in the picker.

                                                                *:GripQuery*
:GripQuery [sql]
  Open a SQL query pad. Optionally pre-fill with a SQL string. Press
  <C-CR> to execute the buffer contents into a grip grid.

                                                                 *:GripSave*
:GripSave [name]
  Save the current query pad content to .grip/queries/{name}.sql.
  Prompts for a name if not provided.

                                                                 *:GripLoad*
:GripLoad [name]
  Load a saved query into the query pad. Shows a picker if no name is
  provided.

                                                              *:GripConnect*
:GripConnect [url]
  Switch database connection. Shows a picker if no argument is provided.
  Reads from .grip/connections.json, g:dbs (DBUI compatible), and
  $DATABASE_URL.

                                                              *:GripExplain*
:GripExplain [sql]
  Query Doctor: translates EXPLAIN plans into plain-English health
  checks. Shows severity (SLOW/WARN/OK), proportional cost bars,
  and actionable suggestions (e.g. "add an index on the filtered
  column"). Works across PostgreSQL, MySQL, DuckDB, and SQLite.

                                                          *:GripProperties*
:GripProperties [table]
  Show full table properties in a floating window. Includes columns with
  types, primary keys, foreign keys, indexes, row estimate, and table size.
  Uses the current grip table if no argument is provided.

                                                              *:GripProfile*
:GripProfile [table]
  Open a data profiling report showing per-column statistics with Unicode
  sparkline distributions. Shows completeness, cardinality, min/max, and
  top values. Adapts to narrow terminals with a stacked layout.

                                                                 *:GripAsk*
:GripAsk [question]
  Generate SQL from a natural language question using AI. Schema context
  is auto-assembled from database metadata. Result appears in the query
  pad for review before execution.

  Providers: Anthropic, OpenAI, Gemini, Ollama (local).
  Auto-detects from env vars: ANTHROPIC_API_KEY > OPENAI_API_KEY >
  GEMINI_API_KEY > ollama fallback. Override with setup({ai={...}}).

                                                              *:GripHistory*
:GripHistory
  Browse query history. Every query executed via :Grip, the query pad,
  and staged DML is recorded in .grip/history.jsonl. Shows a picker
  (telescope/fzf-lua/native) with timestamp and SQL preview. Selecting
  an entry opens it in the query pad for review and re-execution.

                                                                *:GripDiff*
:GripDiff {table1} {table2}
  Compare two tables row-by-row using primary keys. Shows changed cells,
  added rows, and deleted rows with color-coded diff highlighting.
  Auto-selects compact mode on narrow terminals (<120 columns).
  Navigate changes with ]c and [c. Toggle compact/wide with gv.

                                                              *:GripRename*
:GripRename {old_name} [new_name]
  Rename a column in the current table. If new_name is provided, executes
  immediately. If only old_name is provided, prompts for the new name.

                                                              *:GripCreate*
:GripCreate
  Open an interactive table designer. Prompts for table name and columns,
  previews the CREATE TABLE DDL, and asks for confirmation before executing.

                                                                *:GripDrop*
:GripDrop [table]
  Drop a table with typed confirmation (must type table name to confirm).
  Detects foreign key dependents and offers CASCADE if they exist.

  j / k           Move between rows
  h / l           Move cursor within row
  w / b           Next / previous column
  Tab / S-Tab     Next / previous column
  gg              First data row
  G               Last data row

  p               Paste clipboard value into cell

  u               Undo last edit (multi-level, pops undo stack)
  U               Undo all changes (resets to original state)
  a               Apply all staged changes to database

  e               Set all selected cells in column to same value
  d               Toggle delete on all selected rows

  y               Yank selected cells in column (newline-separated)

SORT / FILTER / PAGINATION ~

  s               Toggle sort on column (ASC then DESC then off)
  S               Stack secondary sort on column
  f               Quick filter by cell value (column = value)

  F               Clear all filters
  gp              Load saved filter preset for current table
  gP              Save current filter as named preset
  X               Reset view (clear sort, filter, and page)
  ]p / [p         Next / previous page
  ]P / [P         Last / first page

FK NAVIGATION ~
                                                            *grip-fk-nav*
  gf              Follow foreign key under cursor to referenced row
  <C-o>           Go back in FK navigation stack

ANALYSIS AND EXPORT ~
                                                           *grip-analysis*

  gS              Column statistics popup

  gR              Table profile (sparkline distributions)
  gD              Diff current table against another table

  gA              AI SQL generation (natural language)

INSPECTION ~

  gc              Copy staged SQL to clipboard

  0               Unpin all (or first column if none pinned)

7. EDITING                                          *grip-editing-workflow*

Grip uses a staging model for edits. Changes are staged in memory and
color-coded in the grid before being applied to the database.

Color coding:
  Blue ............. Modified cell (staged change)
  Red .............. Row marked for deletion
  Green ............ Newly inserted row

Workflow:
  1. Edit cells with `e`, set NULL with `n`, paste with `p`
  2. Insert rows with `o`, mark deletions with `d`
  3. Preview staged SQL with `gs` or live preview with `gl`
  4. Apply all changes with `a` (wraps in BEGIN/COMMIT transaction)
  5. Undo individual edits with `u` or reset all with `U`

All DML is wrapped in a transaction. If any statement fails, the entire
transaction is rolled back and your staged changes are preserved so you
can fix the value and retry.

Multi-level undo: every edit operation pushes the previous state onto a
50-deep undo stack. Press `u` repeatedly to step back through your edit
history. Press `U` to reset to the original state in one step.

Read-only mode is auto-detected when no primary key exists on the table
(grip needs PKs for WHERE clauses in UPDATE/DELETE statements).

8. SORT, FILTER, PAGINATE                           *grip-sort-filter-page*

Sort ~
Press `s` on a column header to cycle through ASC, DESC, and off. The
query is re-run with an ORDER BY clause. Press `S` to stack a secondary
sort (multi-column sorting).

Filter ~
Press `f` on a cell to quick-filter by that value (column = value).
Press `<C-f>` to enter a freeform WHERE clause. Multiple filters are
AND-ed together. Press `F` to clear all filters.

Filter presets ~
Press `gP` to save the current active filters as a named preset for the
current table. Presets are stored in `.grip/filters.json` keyed by table
name. Press `gp` to load a saved preset. The picker supports telescope,
fzf-lua, and vim.ui.select.

Paginate ~
Results are paginated (default 100 rows per page). The status line shows
the current page and total. Press `]p`/`[p` to navigate pages, or
`]P`/`[P` to jump to the last/first page.

Press `X` to reset all sort, filter, and pagination state at once.

==============================================================================
9. FOREIGN KEY NAVIGATION                               *grip-fk-nav-full*

Press `gf` on a cell that contains a foreign key value to navigate to the
referenced row in the target table. A new grid opens showing just that
row. Press `<C-o>` to go back. The title bar shows a breadcrumb trail
of your navigation path.

FK metadata is fetched automatically via information_schema (PostgreSQL,
MySQL, DuckDB) or PRAGMA foreign_key_list (SQLite).

10. SCHEMA BROWSER                                          *grip-schema*

Toggle with `:GripSchema` or `go` from a grid buffer. Opens a left sidebar
showing all tables and views for the current connection.

Schema browser keymaps:
  <CR>            Open table in grid (or open parent table from column)
  l / zo          Expand table to show columns
  h / zc          Collapse table
  L               Expand all tables
  H               Collapse all tables
  /               Filter tables by name
  r               Refresh schema (re-fetch table list)
  gT              Open table picker
  gQ              Open query pad
  D               Drop table (with typed confirmation)
  +               Create new table (interactive designer)
  q / <Esc>       Close sidebar

Columns show type abbreviations (int, text, bool, vc(255), etc.) and
markers for primary keys and foreign keys.

==============================================================================
11. QUERY PAD                                             *grip-query-pad*

Open with `:GripQuery` or `gQ` from a grid buffer. Creates a scratch SQL
buffer with filetype=sql (enables vim-dadbod-completion if installed).

Query pad keymaps:
  <C-CR>          Execute buffer contents into a grip grid (normal/insert)
  <C-CR>          Execute visual selection into a grip grid (visual mode)
  <C-s>           Save query with :GripSave

The query pad sets b:db to the current connection URL, which enables
vim-dadbod-completion for table and column name auto-completion.

==============================================================================
12. SAVED QUERIES                                      *grip-saved-queries*

Save queries to project-local .grip/queries/ directory:

  :GripSave report        saves to .grip/queries/report.sql
  :GripLoad report        loads .grip/queries/report.sql
  :GripLoad               shows picker of saved queries

The project root is detected by walking up from the current directory
looking for .git or .grip directories.

13. CONNECTIONS                                          *grip-connections*

`:GripConnect` manages database connections from multiple sources:

  .grip/connections.json     project-local connection profiles
  g:dbs                      vim-dadbod-ui compatible table
  $DATABASE_URL              environment variable
  g:db / b:db                vim-dadbod variables

Add a new connection: >
  :GripConnect postgresql://user:pass@localhost/mydb
<
Or run `:GripConnect` with no arguments to pick from known connections
or add a new one interactively.

Connections are deduplicated by URL across all sources.

==============================================================================
14. EXPORT                                                   *grip-export*

Press `gE` in a grid to export the current view. Six formats are
available:

  CSV .............. RFC 4180 compliant, quoted fields
  TSV .............. Tab-separated values
  JSON ............. Array of objects
  SQL INSERT ....... INSERT statements for the current table
  Markdown ......... GitHub-flavored markdown table
  Grip Table ....... Box-drawing table matching the grid style

All formats are copied to the system clipboard.

Additional yank commands:
  y ................ Yank single cell value
  Y ................ Yank entire row as CSV
  gY ............... Yank entire table as CSV

15. SCHEMA OPERATIONS                                            *grip-ddl*

Table properties ~
Press `gI` in a grid or run `:GripProperties` to open a floating window
showing full table metadata: columns with types, primary keys, foreign
keys, indexes, estimated row count, and table size.

Properties view keymaps:
  R               Rename column under cursor
  +               Add new column (prompts for name, type, default)
  x               Drop column under cursor (typed confirmation)
  gI              Refresh properties
  q / <Esc>       Close

Column rename ~
Press `R` in the properties view or run `:GripRename old_name new_name`.
Previews the ALTER TABLE DDL and asks for confirmation before executing.

Column add/drop ~
Press `+` in properties to add a column. Prompts for column name, data
type, and optional default value. Previews the DDL before executing.
Press `-` in properties to drop a column. Requires typing the column
name to confirm (destructive operation).

Create table ~
Run `:GripCreate` or press `+` in the schema browser. Opens an
interactive column designer that prompts for table name, then each
column (name, type, primary key). Previews the CREATE TABLE DDL and
asks for confirmation.

Drop table ~
Run `:GripDrop` or press `D` in the schema browser. Checks for foreign
key dependents and offers CASCADE if they exist. Requires typing the
table name to confirm.

16. ADAPTERS                                              *grip-adapters*

Grip supports four database adapters. The adapter is selected automatically
based on the connection URL scheme.

  postgresql:// .... PostgreSQL via psql CLI
  sqlite: .......... SQLite via sqlite3 CLI
  mysql:// ......... MySQL/MariaDB via mysql CLI
  duckdb: .......... DuckDB via duckdb CLI

Each adapter implements: query, execute, get_primary_keys, get_column_info,
get_foreign_keys, get_indexes, get_table_stats, list_tables, and explain.

File-as-table: `:Grip /path/to/data.parquet` routes through DuckDB to
query Parquet, CSV, JSON, NDJSON, and XLSX files directly. No database
connection is needed for file queries.

Remote file querying: `:Grip https://example.com/data.csv` opens remote
files via DuckDB's httpfs extension. The extension is loaded automatically.
HTTP queries use a 30-second timeout. Supported formats are the same as
file-as-table. All remote queries open in read-only mode.

17. PUBLIC API                                                  *grip-api*

                                                          *grip.setup()*
grip.setup({opts})
  Optional configuration override. Called automatically with defaults by
  the plugin loader.

  Parameters:
    limit         (number)  Default row limit. Default: 100
    max_col_width (number)  Max display width per column. Default: 40
    timeout       (number)  Query timeout in ms. Default: 10000

                                                           *grip.open()*
grip.open({arg}, {url}, {opts})
  Open a table, run a query, or open a file.

  Parameters:
    arg   (string)  Table name, SQL query, or file path
    url   (string)  Connection URL (optional, uses current connection)
    opts  (table)   Optional: { reuse_win = winid }

                                                     *grip.open_smart()*
grip.open_smart()
  Auto-detects context from the current buffer:
    - DBUI SQL buffer: opens that table, reuses dbout window
    - dbout result buffer: traces back to source table
    - Normal buffer: uses word under cursor as table name
